### User Story I:
As a CS student Christy
needs a server on which she can
so that she can learn practical Computer Networking.

#### Acceptance Criteria:
**Virtual environment** - On Project Jorgan, only Christy has access to a part of the server dedicated to her. She can manipulate this part of the server freely without having an effect on the entire server. Also, nobody else can access Christy's virtual environment unless she gives permission. The same goes for Christy trying to access somebody else's virtual environment. <br> 
**SSH connection** - Christy is able to safely connect to Project Jorgan through an SSH connection. <br> 
**UNIX terminal** - Once connected, Christy is able to run any command which can be run on a UNIX/Linux terminal. <br> 


### User Story II:
As a CS student Christy
wants to host a personally created portfolio website (react/node.js) so that she can let possible employers become aware of her skills.

#### Acceptance Criteria:
**Back end server hosting** - Christy is able to successfully start a node express back-end server and it continues to be hosted even after Christy disconnects from Project Jorgan. <br> 
**Front end server hosting** - Christy is able to successfully start a react.js front-end server and it continues to be hosted even after Christy disconnects from Project Jorgan. <br>
