 ### Product vision
 Project Jorgan aims to developa a personal server hosting services while keeping the user's data in their control. Unlike services like Google which process personal data in order to use their services for free.

 As such, project Jorgan should be able to rival basic services of Google Cloud such as:
 - Google Images
 - Google Docs
 - Google Cloud

 The first two are kind of self explanatory, cloud image storage and cloud document storage. <br> The vision of the third one aims to support a remote development server which can be used to host websites, online video game servers and other services.

 ### Skills
 For the development of this project,  experts are needed in the following areas:
 - Hardware - able to select suitable hardware for a server and build it 
 - Networking - knowledge in dealing with SSH connections; hosting for websites, online video games, etc.;
 - Docker - kinda part of the above, but felt that it should be mentioned here
 - UI - desired but might not be necessary if using open-source solutions for some of the services

 ### Sprint duration
 Each sprint will last 2 to 4 weeks, depending on the tasks it will cover.
 
 ### Metrics
 The progress will be assessed in the following way. Since our solution mainly focuses on the hardware solution, we will be using functionality and unit testing to test whether a given bullet point in the product backlog is functional. If it passes all the unit tests, it should be considered done. Also this project will have no measure for the tasks which are in progress, other than testing smaller sub-functionalities. For example, consider the Image Cloud services, where sub-functionalities could be a successful upload of image and successful download of image.

 ### Product backlog
 Task - success criteria (ommited if it's trivial):
 - Select suitable hardware solutions
 - Select suitable open source software solutions
 - Server setup on LAN - server can be from the same LAN
 - Server setup globally - server can be accessed from any device (from a browser)
 - SSH connection setup - server can be accessed through an SSH-like tool utilizing SSH connection
 - Cloud Image services - storing, viewing, sharing images with a satisfactory UI designed for it
 - Website hosting - server can host up to 3 (low traffic) websites
